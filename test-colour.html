Primary Brand Colors

Core brand colors that define the visual identity

Color Token Usage & Context Accessibility
Navy Primary
#0e3a6c
 --color-navy-500 Primary brand color.
Use for: Page headers, primary buttons, navigation elements, form labels, main headings (H1-H4). This is the dominant brand color that establishes authority and professionalism. 		AAA on white
Contrast ratio: 8.7:1
Brand Red
#a30134
 --color-red-500 Call-to-action and interactive elements.
Use for: Links, CTA buttons, required field indicators (*), interactive elements. Distinct from error red to avoid confusion between clickable elements and validation errors. 		AA on white
Contrast ratio: 6.2:1
Charcoal
#39444f
 --color-charcoal-500 Primary text color.
Use for: Body text, paragraph content, input text, helper text. Provides excellent readability while being softer than pure black. 		AAA on white
Contrast ratio: 9.8:1
Focus Orange
#ff9900
 --color-focus-500 Global focus indicator.
Use for: Focus rings on all interactive elements (buttons, inputs, links). High-visibility color that ensures accessibility compliance for keyboard navigation. 		AA on white
Focus indicator only
Brand Pink
#ffe3ea
 --color-pink-500 Occasional brand accent.
Use for: Special promotions, highlighted content, light backgrounds for error states. Sparingly used brand color for specific campaigns or accent elements. 		Background only
Use with dark text

Status & Feedback Colors

Colors that communicate system states and user feedback

Color Token Usage & Context Variants Available
Success Teal
#007d85
 --color-success-500 Positive feedback and completed states.
Use for: Success messages, completed forms, confirmed actions, positive notifications. Professional teal that conveys reliability without being overwhelming. 		200 (light bg), 400, 500 (primary), 600 (hover)
Warning Orange
#b75b00
 --color-warning-500 Caution and attention-needed states.
Use for: Warning messages, unsaved changes, attention-required notifications. Draws attention without creating alarm. Also used in focus states. 		200 (light bg), 400, 500 (primary), 600 (hover)
Error Red
#eb0000
 --color-error-500 Validation errors and system failures.
Use for: Form validation errors, system error messages, failed operations. Distinct bright red that clearly indicates something went wrong. 		200 (light bg), 400, 500 (primary), 600 (hover)
Destructive Red
#d92b2b
 --color-destructive-500 Irreversible actions and deletions.
Use for: Delete buttons, remove actions, permanent destructive operations. Different from error red to distinguish between validation issues and destructive actions. 		200 (light bg), 400, 500 (primary), 600 (hover)

Color Usage Guidelines

✅ Do

  • Use Navy for all headings and primary actions
  • Use Brand Red only for clickable elements (links, CTAs)
  • Use distinct Error Red for validation messages
  • Use Success Teal for positive feedback
  • Always include focus states with Focus Orange
  • Use lighter variants (200) for backgrounds

❌ Don't

  • Mix Error Red and Brand Red in the same context
  • Use bright colors for large background areas
  • Use Pink as a primary color - it's accent only
  • Skip focus indicators on interactive elements
  • Use colors that don't meet accessibility standards
  • Use Destructive Red for general error messages
